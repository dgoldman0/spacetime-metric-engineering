# Stage 2 beta075 matter-action feasibility worklog

Status: committed skeleton-battery checkpoint. Continue using this as the
matter-action feasibility worklog as the next narrow heavy-duty support-
reservoir tests are developed.

## Working Question

Can the beta075 endpoint covariant tensor be generated by a disciplined
matter-action, constitutive field equation, or actuator-coupled effective
source model without adding hidden live support, superluminal transport,
divergent coefficients, or broad nonlocal exchange?

## Starting Target

The target is not the full active-rail geometry. The target is the frozen
beta075 endpoint medium after the covariant identity audit:

```text
T_mu_nu = rho u_mu u_nu + p_l s_mu s_nu + pOmega Q_mu_nu
          + u_mu q_nu + u_nu q_mu

u = n_ADM
s = e_l
q = j_l s
Q = g + u u - s s
```

Known diagnostic constraints:

```text
ADM projection closure: machine precision on baseline and dense meshes
heat/current boost-to-flux-frame: subluminal on baseline and dense meshes
mixed tensor eigenstructure: real eigenvalues on baseline and dense meshes
divergence/exchange: localized to endpoint/support masks, with small stencil halo
live active medium rows: zero
```

## Claim Boundary

This worklog may develop a source-family story and necessary-condition screens.
It must not claim a final matter theorem, final Lagrangian, global horizon
theorem, semiclassical/RSET proof, or broad beta/V robustness result.

## Candidate Families

### Scalar-only

Prior expectation: ruled out or retained only as a subcomponent.

Reason to test:

```text
Earlier source-class screens already disfavor single canonical/phantom scalar
models. Record the algebraic obstruction clearly so the matter-action story
does not regress to an attractive but too-small source family.
```

Findings:

```text
Initial data pull keeps scalar-only ruled out.

Baseline source-class screen:
  canonical scalar compatible volume fraction: 0.379515
  phantom scalar compatible volume fraction:   0.382312

Dense source-class screen:
  canonical scalar compatible volume fraction: 0.335594
  phantom scalar compatible volume fraction:   0.407455

Interpretation:
  A single scalar sign choice cannot span the endpoint tensor. The obstruction
  is not a thin-grid artifact; dense refinement keeps both scalar branches far
  below the 0.95 compatibility screen. Scalar fields may still be subcomponents
  in a multi-field construction, but scalar-only should stay dead.

Algebraic read:
  For a minimally coupled scalar with time/radial gradient, the radial block is
  boost-diagonalizable and the signs of rho+p_l and p_l-pOmega are tied to the
  scalar kinetic sign. The endpoint tensor violates a single global choice of
  those signs across most of the source support, and the pre-regulator
  flux-dominant burden also exceeds what scalar-only can carry.
```

### Elastic/solid anisotropic medium

Prior expectation: plausible for radial/angular stress split, incomplete for
current unless coupled to a transport field.

Reason to test:

```text
Can naturally encode direction-dependent stress and finite support. Must explain
radial heat/current and localized exchange current rather than only pressures.
```

Findings:

```text
Initial read: plausible for pressure anisotropy, incomplete for current.

Feasibility supports:
  The endpoint source is finite-width and non-live.
  The angular watch is carried as an internal anisotropic response.
  Dense refinement decreases the fitted angular-response coefficient and the
  covariant active divergence norm.

Risk:
  A solid/elastic action naturally gives direction-dependent stress, but it
  does not automatically provide radial heat/current q_mu or the regulator
  cushion. It probably needs a transport/vector companion or an open-system
  coupling. Pure elastic pressure support alone is therefore likely too narrow.
```

bounded and localized.

Findings:

```text

  the measured tensor if allowed enough functions. The risk is explanatory

Interpretation:

  tensor-form or exchange-current tests. Any multi-field proposal must have a
  strict coefficient/support audit so it does not reintroduce the overfitting
  concern that the covariant tensor audit was meant to reduce.
```

## First Necessary Tests

```text
1. Degree-of-freedom count: can the family express u, s, q, p_l, pOmega,
   regulator cushion, and localized exchange?
2. Tensor-form match: does the family naturally produce the measured
   anisotropic heat/current tensor?
3. Causality: does the heat/current mode remain subluminal under the measured
   boost margins?
4. Angular response: can the family carry the non-live pOmega watch internally?
5. Exchange current: can nonzero divergence be balanced by support/actuator
   coupling localized to existing masks?
6. No hidden support: no live rows, extra broad support, singular coefficients,
   or extra spatial directions unless explicitly justified.
```

## Provisional Positive Ansatz A: Entrained Director Medium With Support Exchange

Status: first positive family to develop.

Minimal field content:

```text
u^mu: constrained timelike medium velocity
s^mu: constrained spacelike radial/support director
r: radial transport/current amplitude, q^mu = q s^mu
h_reg: regulator/enthalpy cushion controlling rho + p_l - 2 |j_l|
chi_Omega: internal angular-response state controlling pOmega - p_l
J_support^nu: localized exchange current with support/actuator plant
```

Target stress form:

```text
T_endpoint_mu_nu =
  rho u_mu u_nu
  + p_l s_mu s_nu
  + pOmega (g_mu_nu + u_mu u_nu - s_mu s_nu)
  + u_mu q_nu + u_nu q_mu
```

Interpretive action sketch:

```text
S_total = S_medium[u, s, r, chi_Omega, constraints; g]
          + S_support[actuator/support fields; g]
          + S_exchange[medium, support; g]

nabla_mu T_medium^{mu nu} = J_support^nu
nabla_mu (T_medium^{mu nu} + T_support^{mu nu}) = 0
```

Why this is the current lead:

```text
1. It names the measured preferred radial direction instead of hiding it.
2. It can carry the symmetric heat/current term as entrainment between the
   medium clock and radial director/current.
3. It can treat pOmega as an internal anisotropic response rather than a new
   angular source component.
4. It has a natural place for the observed time/radial support exchange.
5. It remains compact enough to be falsifiable by coefficient, support,
   hyperbolicity, and live-leakage screens.
```

Immediate feasibility questions:

```text
1. Can the entrainment/transport sector preserve the dense support-edge
   heat-current margin?
2. Can chi_Omega reproduce the negative angular-inertia watch without adding
   an angular exchange channel?
3. Can S_exchange be localized to the existing reset/support masks and produce
   both power and radial-momentum exchange?
4. Can total conservation be restored with an explicit support stress tensor
   without changing the live packet ledger?
```

Formal constitutive/action skeleton:

```text
Fields and constraints:
  u^mu: unit timelike medium clock, u_mu u^mu = -1
  s^mu: unit spacelike support/radial director, s_mu s^mu = 1
  u_mu s^mu = 0
  r or q: radial transport amplitude with q^mu = q s^mu
  h_reg: enthalpy/current cushion enforcing |rho + p_l| - 2 |q| > 0
  chi_Omega: internal angular-response variable
  support variables: actuator/reservoir sector carrying J_support^nu

Constitutive stress:
  pOmega = p_l + chi_Omega
  Q_mu_nu = g_mu_nu + u_mu u_nu - s_mu s_nu

  T_endpoint_mu_nu =
      rho u_mu u_nu
    + p_l s_mu s_nu
    + pOmega Q_mu_nu
    + u_mu q_nu + u_nu q_mu

Radial causality / Type-I regulator:
  h = rho + p_l
  H = |h|
  heat/current ratio = 2 |q| / H
  h_reg = H - 2 |q|
  require h_reg > 0 and heat/current ratio < 1
  convenient bounded variable: 2 q / H = tanh(psi), with finite psi

Exchange equation:
  nabla_mu T_endpoint^{mu nu} = J_endpoint^nu
  nabla_mu T_support^{mu nu} = -J_endpoint^nu
  nabla_mu (T_endpoint^{mu nu} + T_support^{mu nu}) = 0

Action-level read:
  S_total = S_medium[u, s, q, h_reg, chi_Omega, constraints; g]
          + S_support[actuator/reservoir fields; g]
          + S_exchange[medium, support; g]

  The current data supports this only as an effective/open-system skeleton.
  It does not support a closed endpoint-only matter action.
```
